### P2-QR: QR/Slot On-Lot Capture System

**Status:** ‚òÖ‚òÖ‚òÖ‚òÖ‚òÖ PHASE 2 ANCHOR
**Quadrant:** ‚≠ê Quick Win (Medium Effort, High Impact)
**Engine:** ACQUISITION

| Attribute | Detail |
|-----------|--------|
| **Problem Addressed** | Single attendant cannot capture multiple simultaneous visitors; prospects leave without contact captured; 0% capture rate for browsers |
| **Solution** | 20 fixed slot-based QR codes per lot, backend maps slot to current inventory, self-service scan + intake form |
| **Impact** | HIGH - Capture leads from ALL visitors, not just those attendant engages; enable follow-up for browsers |
| **Effort** | MEDIUM - 1-2 weeks for core implementation |
| **Investment** | Part of Phase 2 ($5,000) |
| **Dependencies** | Phase 1 complete (database, form infrastructure) |
| **IP Risk** | MEDIUM - Proprietary capture methodology |
| **Timeline** | Week 1-2 |

#### How It Works

```
Customer walks lot ‚Üí Scans QR on shed ‚Üí
    ‚Üí QR links to slot number (not model ID)
    ‚Üí Backend maps slot to current inventory
    ‚Üí Customer sees shed details + intake form
    ‚Üí Scan event logged (timestamp, slot, session)
    ‚Üí Lead captured even if customer leaves
```

#### Deliverables

| Component | Description | Status |
|-----------|-------------|--------|
| QR Code Generation | 20 codes per lot (60 total for 3 lots) | ‚¨ú Pending |
| Slot Mapping Backend | Firestore collection mapping slots ‚Üí inventory | ‚¨ú Pending |
| Scan Event Tracking | Log all scans with timestamp, slot, session ID | ‚¨ú Pending |
| Mobile Landing Page | Shed details + intake form per slot | ‚¨ú Pending |
| Weather-Resistant QR Signage | Physical QR codes for lot deployment | ‚¨ú Pending |

# Meeting Folder Template

## Quick Setup

To create a new meeting folder:

```bash
# Set meeting details
MEETING_DATE="YYYY-MM-DD"
MEETING_NAME="meeting-name"

# Create folder structure
mkdir -p "claude-code-os-implementation/01-executive-office/internal-business-meetings/meetings/${MEETING_DATE}-${MEETING_NAME}/01-raw"
mkdir -p "claude-code-os-implementation/01-executive-office/internal-business-meetings/meetings/${MEETING_DATE}-${MEETING_NAME}/02-processed"
mkdir -p "claude-code-os-implementation/01-executive-office/internal-business-meetings/meetings/${MEETING_DATE}-${MEETING_NAME}/03-integrated"

# Create placeholder files from templates
cp claude-code-os-implementation/01-executive-office/internal-business-meetings/templates/* \
   claude-code-os-implementation/01-executive-office/internal-business-meetings/meetings/${MEETING_DATE}-${MEETING_NAME}/02-processed/
```

## File Purposes

### 01-raw/ (Capture Phase)

**audio.m4a / audio.mp3 / audio.wav**
- Original meeting recording
- Supports: .m4a, .mp3, .wav, .mp4
- Auto-processed by Whisper

**transcript.md**
- Auto-generated by Whisper transcription
- Full verbatim transcript
- Time-stamped if available

**fathom-export.md** (optional)
- Fathom AI meeting summary
- Pre-processed highlights
- Use when available to speed up processing

**raw-notes.md**
- Manual notes taken during meeting
- Quick captures of decisions/action items
- Helps guide processing

### 02-processed/ (Extraction Phase)

**overview.md**
- Meeting metadata (date, attendees, duration)
- Key topics discussed (3-5 bullets)
- Overall purpose and outcome
- Links to related projects/docs

**action-items.md**
- All action items with owners
- Priority levels (ðŸ”´ ðŸŸ¡ ðŸŸ¢)
- Deadlines or target dates
- Context for each item

**decisions.md**
- Key decisions made
- Context and reasoning
- Implications and next steps
- Owner for implementation

**roadmap-additions.md**
- Strategic items for roadmap
- Business value assessment
- Priority level (P0-P5)
- Timeline estimates

### 03-integrated/ (Integration Phase)

**system-updates.md**
- Log of where action items were added
- Which files were updated
- What communications were sent
- Verification checklist

## Processing Workflow

### Phase 1: Raw Capture
1. Drop audio file in `01-raw/audio.*`
2. Take notes in `01-raw/raw-notes.md` during meeting
3. Add Fathom export to `01-raw/fathom-export.md` (if available)

### Phase 2: Transcription
1. Whisper auto-transcribes audio â†’ `01-raw/transcript.md`
2. Or manually trigger: `bash audio-processing-tools/whisper-local/batch-processor.sh /path/to/audio.m4a`

### Phase 3: Processing
1. Read transcript + raw notes
2. Use Claude to extract structured data
3. Fill in files in `02-processed/`

### Phase 4: Integration
1. Add urgent items to daily plan
2. Add important items to weekly plan
3. Add strategic items to active-items.md
4. Update roadmap files
5. Document in `03-integrated/system-updates.md`

## Common Meeting Types

### Partner Huddle / Weekly Sync
**Focus:** Strategy, pipeline, blockers
**Attendees:** All partners
**Typical outputs:** Strategic decisions, priority changes, action assignments

### Deal Review (with Linh)
**Focus:** Specific deals, client relationships
**Attendees:** You + Linh (+ Mikael sometimes)
**Typical outputs:** Deal strategy, pricing decisions, next client actions

### Pipeline Planning (with Mikael)
**Focus:** Outreach, new leads, partnerships
**Attendees:** You + Mikael
**Typical outputs:** Outreach priorities, target lists, relationship updates

### Client Discovery / Diagnostic Call
**Focus:** Understanding client needs
**Attendees:** You + Client (+ partner who brought lead)
**Typical outputs:** Project scope, pricing estimate, next steps

### Project Review
**Focus:** Active project status
**Attendees:** Varies by project
**Typical outputs:** Timeline updates, technical decisions, resource needs

## Naming Conventions

### Meeting Folder Names
Format: `YYYY-MM-DD-meeting-type`

Examples:
- `2025-12-10-partner-huddle`
- `2025-12-11-deal-review-linh`
- `2025-12-12-client-discovery-plotters`
- `2025-12-15-weekly-sync`

### Audio File Names
Format: `audio.{extension}`

Supported:
- `audio.m4a` (preferred)
- `audio.mp3`
- `audio.wav`

## Integration Checklist

After processing each meeting:

- [ ] **Transcript created:** `01-raw/transcript.md` exists
- [ ] **Overview complete:** `02-processed/overview.md` filled in
- [ ] **Action items extracted:** `02-processed/action-items.md` complete
- [ ] **Decisions documented:** `02-processed/decisions.md` filled in
- [ ] **Roadmap updated:** `02-processed/roadmap-additions.md` if applicable
- [ ] **Urgent items added:** To today/tomorrow's daily plan
- [ ] **Important items scheduled:** In this week's plan
- [ ] **Strategic items tracked:** In active-items.md
- [ ] **Integration logged:** `03-integrated/system-updates.md` complete
- [ ] **Partners notified:** Action item owners informed

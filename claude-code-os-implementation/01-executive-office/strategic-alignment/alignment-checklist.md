## üîß Realignment Protocol

### When Misalignment Detected

**Step 1: PAUSE**
- Stop all current work
- Block 30 minutes for realignment

**Step 2: REVIEW**
- Read OBG out loud
- Review P1 priorities
- Check current task list

**Step 3: IDENTIFY**
- What caused misalignment?
- How long have I been drifting?
- What needs to change?

**Step 4: CORRECT**
- Kill misaligned tasks
- Reschedule calendar to P1 focus
- Set new THE ONE THING
- Update kill list

**Step 5: PROTECT**
- What will prevent future drift?
- What boundaries to set?
- What to automate/delegate?

**Step 6: RESUME**
- Start with P1 task
- Set next alignment check

## ‚úÖ Alignment Best Practices

### Daily Practices
1. Start day with alignment check
2. End day with alignment review
3. Visible OBG reminder (screensaver, note, etc.)
4. THE ONE THING on whiteboard
5. Kill list visible

### Weekly Practices
1. Sunday evening: Next week planning + alignment
2. Wednesday midweek: Alignment checkpoint
3. Friday: Week review + alignment score
4. Decline meetings that don't serve P1/P2
5. Time block for P1 work

### Monthly Practices
1. Full alignment audit last Sunday
2. Pattern analysis for strategic insights
3. Priority level review
4. Resource reallocation if needed
5. Next month OBG connection plan

### Quarterly Practices
1. OBG achievement assessment
2. New OBG definition
3. Strategic priority reset
4. System optimization
5. Lessons documentation

   - Strategic value (internal tools third)

**Output:** Prioritized list of projects to review/test/deliver

---

### Phase 2: Review & Test (Systematic Approach)
**Goal:** Evaluate what's been built and assess quality

**For Each Project:**
1. [ ] **Manual testing**
   - Does it work as expected?
   - What breaks or fails?
   - Edge cases handled?
   - User experience acceptable?

2. [ ] **Code review** (if needed)
   - Quality of AutoClaude's output
   - Security considerations
   - Performance issues?
   - Documentation adequate?

3. [ ] **AutoClaude evaluation**
   - Did it follow the roadmap?
   - Quality vs. manual coding?
   - What would I do differently?
   - Trust level for future tasks?

**Output:** Pass/fail + gap list for each project

---

### Phase 3: Gap Closure (Get to Shippable)
**Goal:** Fix what's broken, fill what's missing

**Actions:**
1. [ ] For each project, create gap list:
   - Critical (must fix to ship)
   - Important (should fix before handoff)
   - Nice-to-have (defer)

2. [ ] Prioritize gap closure:
   - Fix critical gaps first
   - Use AutoClaude for repetitive fixes
   - Manual intervention where needed

3. [ ] Re-test after fixes
   - Verify gaps closed
   - No new issues introduced
   - Meets delivery standard

**Output:** Shippable projects ready for handoff

---

### Phase 4: Delivery (Ship It)
**Goal:** Get it into client/user hands

**Actions:**
1. [ ] **Prepare for handoff:**
   - Documentation complete?
   - Demo/walkthrough ready?
   - Support plan defined?

2. [ ] **Deliver to client:**
   - Schedule handoff meeting
   - Walk through functionality
   - Gather feedback
   - Address immediate issues

3. [ ] **Collect payment/value:**
   - Invoice if client project
   - Use for demo if sales asset
   - Implement if internal tool

**Output:** Delivered project, payment collected or value realized

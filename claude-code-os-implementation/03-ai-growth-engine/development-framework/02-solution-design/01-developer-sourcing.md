## The Golden Rules

1. **Never hire without seeing similar work**
2. **Always negotiate rates down 20-30%**
3. **Fixed price > hourly when possible**
4. **Multiple developers > single dependency**
5. **Pay only after quality gates pass**

---

## Emergency Protocols

### Developer Goes Dark
1. 24-hour grace period
2. Attempt all communication channels
3. Engage backup developer
4. Take over with internal resources
5. Document for legal if needed

### Quality Issues
1. Specific feedback with examples
2. One chance to fix
3. Bring in senior developer to assist
4. Take over if not resolved
5. Partial payment only

### Timeline Slipping
1. Daily check-ins
2. Reduce scope if needed
3. Bring in additional help
4. Set hard deadline
5. Financial penalty if missed

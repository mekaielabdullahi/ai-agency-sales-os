# Project Kickoff Checklist

## Pre-Kickoff (Before First Day)

### Administrative
- [ ] Contract signed
- [ ] Initial payment received
- [ ] Project folder created
- [ ] Slack/Discord channel created
- [ ] Repository initialized
- [ ] Project board setup

### Requirements
- [ ] Requirements document reviewed
- [ ] Scope boundaries defined
- [ ] Success criteria documented
- [ ] Milestones agreed
- [ ] Timeline confirmed
- [ ] Budget approved

### Resources
- [ ] Developer assigned/sourced
- [ ] Architect time blocked
- [ ] Tools/services provisioned
- [ ] Development environment ready
- [ ] Staging environment planned

---

## Day 1: Kickoff

### Morning: Setup (2 hours)
- [ ] Team introduction call
- [ ] Repository access granted
- [ ] Communication channels confirmed
- [ ] Development environment verified
- [ ] CI/CD pipeline configured

### Afternoon: Planning (2 hours)
- [ ] Requirements walkthrough
- [ ] Technical approach confirmed
- [ ] First milestone tasks defined
- [ ] Daily standup time agreed
- [ ] First week plan created

### End of Day
- [ ] Kickoff summary sent to client
- [ ] Team has everything needed
- [ ] First tasks assigned
- [ ] Tomorrow's goals clear

---

## Week 1: Foundation

### Day 2-3: Architecture
- [ ] Architecture documented
- [ ] Database schema designed
- [ ] API structure defined
- [ ] Security model planned
- [ ] Infrastructure decided

### Day 4-5: Development Start
- [ ] Core structure built
- [ ] Basic functionality working
- [ ] Tests being written
- [ ] Daily updates happening
- [ ] Progress visible to client

### End of Week 1
- [ ] Week 1 report sent
- [ ] Client feedback incorporated
- [ ] Next week planned
- [ ] Risks identified
- [ ] Team rhythm established

---

## Critical Success Factors

### Must Have by End of Week 1
- [ ] Working development environment
- [ ] Code in repository
- [ ] Basic functionality demonstrated
- [ ] Client engaged and responsive
- [ ] Team communicating well
- [ ] Progress tracking active

### Red Flags to Watch
- [ ] Requirements still unclear
- [ ] Technical blockers unresolved
- [ ] Developer not communicating
- [ ] Client not responsive
- [ ] Scope creep starting
- [ ] Timeline already slipping

---

## Communication Checklist

### Daily
- [ ] Developer standup update
- [ ] Progress logged
- [ ] Blockers identified
- [ ] Commits pushed

### Tri-Weekly
- [ ] Client update sent
- [ ] Demo if applicable
- [ ] Feedback collected

### Weekly
- [ ] Formal report sent
- [ ] Metrics updated
- [ ] Next week planned
- [ ] Risks reviewed

- [ ] Standards followed

- [ ] All tests passing
- [ ] Code reviewed
- [ ] Documentation current
